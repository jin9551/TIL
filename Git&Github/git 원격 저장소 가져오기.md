# git 원격 저장소 가져오기

가끔 원격 저장소에 있는 다른 팀원들의 브랜치를 가져와야하는 경우가 있다.
이럴때 사용하는 방법이다


### 원격 브랜치 목록 업데이트 및 목록 확인

```
# 원격의 브랜치를 찾지 못해서 발생하는 오류를 해결해준다
git remote update

# 그리고 원격 저장소 브랜치 목록 조회
git branch -r #혹은 git branch -a
```

### 원격 브랜치 가져오기

```
# 원격 저장소의 브랜치 이름 그대로 가져오기
git checkout -t [원격 브랜치 이름] # 예)git checkout -t origin/develop

# 원격 저장소의 브랜치 이름을 변경하여 가져오기
git checkout -b [생성할 브랜치 이름] [원격 브랜치 이름]
```



### 원격 브랜치 참조하기

간혹 수정 내역을 원격 저장소에 push하지는 않지만 해당 원격 저장소의 변경 사항을 참고하기 위해 로컬에 받아서 사용하고 싶은 경우도 있다.

```
git checkout [원격 브랜치 이름]
```

아무런 옵션 없이 원격 저장소의 브랜치를 체크아웃하면 detached HEAD 상태로 소스를 보고 변경해 볼 수 있게 된다. 이 변경 사항들은 commit, push 할 수 없으며 다른 브랜치로 체크아웃하면 사라진다.
