역할(Role)과 정책(Policy)의 주요 차이점은 다음과 같습니다.

## 🔑 IAM 역할 (Role) vs. 정책 (Policy)

---

### 역할 (Role) 🎭

* **정의:** 특정 **권한의 집합**을 가지고 있지만, **특정 사람이나 자격 증명(credential)과 영구적으로 연결되어 있지 않은** AWS ID입니다. 일종의 '임시 권한 컨테이너' 또는 '옷'과 같습니다.
* **핵심 기능:** 어떤 주체(사람, AWS 서비스, 다른 계정의 사용자 등)가 **일시적으로 빌려 입어(assume)** 그 역할에 정의된 권한을 획득할 수 있게 합니다.
* **목적:** 특정 작업을 수행해야 할 때 **필요한 권한만을** 부여하는 데 사용됩니다. 이는 주로 **서비스(EC2, Lambda 등)에 권한을 부여**하거나, **다른 AWS 계정 또는 연동된 사용자에게 임시 접근 권한을 제공**할 때 사용됩니다.

> **비유:** 역할은 특정 직무(예: '재무팀 관리자' 또는 '배트맨 의상')에 필요한 **뱃지(권한)가 모두 들어있는 주머니**이며, 필요할 때 누구나 그 주머니를 잠시 빌려 사용할 수 있습니다.

---

### 정책 (Policy) 📜

* **정의:** AWS 리소스에 대한 **권한을 공식적으로 정의하는 문서**입니다. JSON 형식으로 작성됩니다.
* **핵심 기능:** **무엇을 허용(Allow)하거나 거부(Deny)**할지, **어떤 리소스**에 대해, **어떤 조건**에서 허용할지를 명시합니다.
* **목적:** 이 정책은 **ID(사용자, 그룹, 역할)** 또는 **리소스(S3 버킷, SQS 대기열 등)**에 **부여(Attach)**되어 실제 권한을 부여합니다. 정책이 없으면 역할이든 사용자든 아무것도 할 수 없습니다.

> **비유:** 정책은 "S3 버킷 A에 데이터를 업로드하는 것을 허용한다" 또는 "EC2 인스턴스 B를 종료하는 것을 거부한다"와 같은 **구체적인 규칙이 적힌 계약서**입니다.

---

### 📝 간단 비교 요약

| 구분 | 역할 (Role) | 정책 (Policy) |
| :--- | :--- | :--- |
| **본질** | 권한을 부여받는 **ID의 한 종류** (권한의 집합) | 권한을 **정의하는 문서** (규칙 명세서) |
| **용도** | **주체(Identity)에게 임시 권한을 제공**하거나 AWS 서비스에 권한을 위임 | **실제 권한 내용을 정의**하고 ID 또는 리소스에 연결 |
| **연결 방식** | 정책을 **부여(Attach)**받음 | 사용자, 그룹, 역할에 **부여(Attach)**됨 |

**결론적으로,** **정책(Policy)**은 **무슨 일**을 할 수 있는지를 정의하고, **역할(Role)**은 그 정책을 담아 **누가** 그 일을 할 수 있도록 할지 결정하는 **수단**입니다.

## 🎯 역할(Role)과 정책(Policy)의 관계 요약

| 개념 | 역할 (Role) 🎭 | 정책 (Policy) 📜 |
| :--- | :--- | :--- |
| **기능** | 정책을 **구현**하여 **권한을 부여**하는 **수단** | 리소스에 대한 **실제 규칙**을 **정의**하는 **명세서** |
| **내용** | **어떤 주체**가 이 권한을 사용할 수 있는지 결정 | **어떤 리소스**에 대해 **무슨 작업**을 할 수 있는지 정의 |
| **주체** | **불특정 다수**의 사용자, 서비스, 또는 외부 계정이 **일시적으로 가정**하여 사용 | 역할, 사용자, 그룹 또는 리소스 자체에 **연결**되어 효력을 발휘 |

**정책**은 "이 S3 버킷에 데이터를 쓸 수 있다"는 **규칙**을 정의하고, **역할**은 이 규칙을 담아서 "특정 조건을 만족하는 누구든지(불특정 사용자) 이 규칙에 따라 S3 버킷에 데이터를 쓸 수 있도록" **권한을 위임**하는 용도입니다.

즉, **정책은 권한의 내용**을, **역할은 권한의 위임 방식**을 결정합니다.
