## 🔐 AWS ID 및 액세스 관리(IAM) 핵심 요약

### 1. IAM 개요

* **정의:** **AWS 리소스 및 서비스에 대한 액세스를 관리**할 수 있게 해주는 **무료 AWS 서비스**입니다.
* **기능:** 사용자 및 그룹을 생성 및 관리하고, 다양한 리소스에 대한 접근 권한을 **허용(Allow)**하거나 **거부(Deny)**하도록 설정할 수 있습니다.
* **적용 범위:** IAM 권한은 **글로벌(Global)**합니다. 즉, 한 사용자에게 설정한 접근 권한은 모든 AWS 리전에서 유효합니다.
* **기본 원칙:** 액세스 권한을 설정할 때는 **최소 권한의 원칙(Principle of Least Privilege)**을 따라야 합니다.

---

### 2. IAM을 통한 액세스 설정 방법

AWS는 IAM을 통해 다음 세 가지 방법으로 사용자와 서비스에 대한 액세스를 관리할 수 있도록 돕습니다.

#### 1. IAM 사용자 관리 (Manage Users)
* **정의:** IAM 내에서 생성되며, 개별적인 보안 자격 증명(Security Credentials)이 할당되는 엔티티입니다.
* **권한:** **매우 세부적인 권한 세트**를 부여하여, 특정 사용자가 어떤 서비스에서 어떤 작업을 수행할 수 있는지 제어할 수 있습니다.
* **유형:**
    * **관리자:** AWS 콘솔에 액세스하여 계정을 관리하는 사람.
    * **최종 사용자(End Users):** AWS 클라우드 계정의 콘텐츠에 접근해야 하는 사람.
    * **시스템:** **프로그래밍 방식 액세스(Programmatic Access)**를 통해 AWS 리소스의 데이터에 직접 접근해야 하는 애플리케이션이나 시스템.

#### 2. IAM 역할 관리 (Manage Roles)
* **정의:** 특정 권한을 관리하기 위해 생성되며, **특정 주체(Entity)가 임시로 가정(assume)할 수 있는** 권한 세트입니다.
* **특징:** 역할을 가정한 주체는 **임시 보안 자격 증명(Temporary Credentials)**을 받아 AWS 리소스에 API 호출을 할 수 있습니다.
* **주요 용도:** 한 조직이 개발 환경과 운영 환경을 분리하는 경우처럼, **다른 AWS 계정의 사용자에게 접근 권한을 제공**하는 데 사용됩니다.

#### 3. 연동 사용자 관리 (Manage Federated Users)
* **정의:** 엔터프라이즈의 기존 ID(예: 회사 Active Directory 사용자)가 **별도의 IAM 사용자를 생성할 필요 없이** AWS 클라우드 인스턴스에 액세스하도록 허용하는 방식입니다.
* **구현:** **SAML 2.0**을 지원하는 ID 관리 솔루션을 사용하여 ID 연동(Identity Federation)을 활성화합니다.
* **일상 비유:** Facebook이나 Gmail 계정을 사용하여 다른 온라인 서비스에 가입하는 것과 유사합니다.

---

### 3. IAM의 주요 이점

* **향상된 보안:** 최소 권한 원칙을 적용하여 보안 수준을 높입니다.
* **세부적인 제어:** 사용자 및 서비스에 대한 접근 권한을 매우 상세하게 제어할 수 있습니다.
* **임시 자격 증명 제공:** 역할(Role)을 통해 임시 자격 증명을 부여할 수 있어 보안성이 높습니다.
* **유연한 자격 증명 관리:** 보안 자격 증명을 유연하게 관리할 수 있습니다.
* **외부 ID 시스템 활용:** ID 연동을 통해 외부 ID 시스템(예: 회사 Active Directory)을 활용할 수 있습니다.
* **AWS 서비스 통합:** 다양한 AWS 서비스와 원활하게 통합됩니다.
