
## 🛠️ AWS Systems Manager 핵심 요약

**AWS Systems Manager**는 AWS, 멀티 클라우드 및 하이브리드 환경의 리소스를 안전하게 관리하는 데 도움을 주는 **중앙 집중식 관리 서비스**입니다.

### 1. 주요 역할 및 특징

* **중앙 집중식 제어:** 여러 AWS 시스템에 개별적으로 로그인할 필요 없이, Systems Manager에서 다양한 AWS 서비스에 대한 작업을 시각화하고 운영할 수 있습니다.
* **환경 지원:** 인프라가 **클라우드**, **멀티 클라우드** 또는 **하이브리드 환경**을 사용하는지 여부와 관계없이 관리를 지원합니다.
* **대상 사용자:** IT 인프라가 원활하게 실행되는지 확인하고, 리소스가 내부 규정 준수 정책을 충족하지 못할 경우 알림을 받아야 하는 **IT 관리자**에게 유용한 도구입니다.

### 2. 핵심 기능

* **자동화:** 패치 적용 및 리소스 변경과 같은 프로세스를 **자동화**합니다.
* **이슈 관리 및 해결:** 문제를 신속하게 **식별**하고, 사용자에게 영향을 미치기 전에 해당 문제를 **자동으로 해결**합니다.
* **운영 데이터 중앙 집중화:** 운영 데이터를 한곳에 모아두어 검토하기 쉽도록 **중앙 집중화**합니다.
* **리소스 그룹 관리:**
    * **논리적인 리소스 그룹**을 생성할 수 있습니다.
    * 그룹을 선택하여 **최근 API 활동, 구성 변경, 알림, 운영 경고, 패치 규정 준수 상태** 등 다양한 메트릭을 한눈에 볼 수 있습니다.
    * 필요한 경우 Systems Manager 내에서 리소스 그룹에 대해 **즉각적인 조치**를 취할 수 있습니다.

---

AWS Systems Manager는 IT 인프라의 운영 상태를 파악하고 일상적인 관리 작업을 자동화하여 **운영 효율성**과 **규정 준수**를 높이는 데 중점을 둔 서비스입니다.
