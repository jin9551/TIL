이 자료는 **최소 권한의 원칙 (Principle of Least Privilege)**과 AWS의 **ID 및 액세스 관리 (Identity and Access Management, IAM)**, 그리고 **네트워크 트래픽 제어** 개념을 중심으로 다루고 있습니다.

---

## 🔒 AWS 보안 및 액세스 관리 핵심 요약

### 1. 최소 권한의 원칙 (Principle of Least Privilege)

* **정의:** 리소스를 안전하게 보호하기 위해, 어떤 개체(Entity)가 **자신의 업무를 수행하는 데 필요한 최소한의 접근 권한**만을 제공하고 그 이상은 허용하지 않는 보안 개념입니다.

---

### 2. ID 및 액세스 관리 (IAM) 용어 해설

AWS는 **IAM (Identity and Access Management)** 및 **IAM Identity Center**와 같은 서비스를 통해 액세스 권한 관리를 돕습니다. 핵심 용어는 다음과 같습니다.

| 용어 | 정의 | 예시 및 설명 |
| :--- | :--- | :--- |
| **Identity (ID)** | 리소스에 대한 액세스를 제공받는 **주체**입니다. | 사용자, 사용자 그룹, 역할, AWS 서비스, 애플리케이션 사용자 등 |
| **Workforce Identity** | 조직에 소속된 **인간 사용자** (관리자, 개발자 등)입니다. | 회사의 직원 계정 |
| **Workload (워크로드)** | 비즈니스 가치를 제공하는 **리소스와 코드의 집합**입니다. | 웹 애플리케이션, Amazon EC2 인스턴스 등. AWS 서비스에 데이터 읽기/쓰기 액세스를 요청할 수 있음. |
| **Federated Identity (연동 ID)** | 다른 서비스의 계정을 사용하여 AWS 환경에 로그인하는 방식입니다. | Facebook이나 Google 계정을 사용하여 블로깅 플랫폼에 로그인하는 것과 유사. 별도의 AWS 계정을 만들 필요 없이 여러 시스템에서 ID를 연결함. |
| **SSO (Single Sign-On)** | 단일 인증 토큰으로 여러 시스템에 로그인할 수 있도록 합니다. AWS는 자체 SSO 서비스인 **AWS IAM Identity Center**를 제공합니다. |

---

### 3. IAM 역할 및 정책

* **IAM Identity Center (舊 AWS Single Sign-on):** AWS Organization 내의 모든 AWS 계정과 애플리케이션에 대한 **작업 인력 권한 및 정책을 생성하고 관리하는 중앙 집중식 제어 센터**입니다.
* **IAM Role (IAM 역할):**
    * 특정 권한 정책을 가진 ID입니다. 특정 사람과 연결되지 않고, **필요한 권한을 부여받기 위해 ID가 임시로 '가정(Assume)'하는** 개념입니다.
    * 예를 들어, 샐리(Sally)는 때로는 '관리 보조' 역할, 때로는 '영업 엔지니어' 역할을 가정하여 다른 권한을 가질 수 있습니다.
* **Policy (정책):**
    * ID 또는 리소스가 **무엇을 할 수 있는지/할 수 없는지** 정의합니다.
    * 정책 내의 **Permission (권한)**은 특정 작업이 **허용(Allow)**되거나 **거부(Deny)**되는지 정의합니다.

---

### 4. 네트워크 트래픽 제어

AWS 리소스에 대한 트래픽을 제어하는 두 가지 주요 방법은 **보안 그룹**과 **네트워크 접근 제어 목록 (네트워크 ACL)**입니다.

| 구분 | 보안 그룹 (Security Group) 🚪 | 네트워크 접근 제어 목록 (Network ACL) 🚧 |
| :--- | :--- | :--- |
| **적용 레벨** | **인스턴스 레벨** (예: Amazon EC2 인스턴스) | **서브넷 레벨** (네트워크 내의 네트워크) |
| **상태** | **Stateful (상태 저장)** | **Stateless (상태 비저장)** |
| **트래픽 허용** | **인바운드 허용 시 아웃바운드 자동 허용** (기억함) | **인바운드와 아웃바운드를 각각 별도로 정의**해야 함 |
| **거부(Deny)** | 명시적 **거부 불가** (Explicit Deny 불가) | 명시적 **거부 가능** (Explicit Deny 가능) |
| **기본 규칙** | 인바운드: **모두 차단** (Blocked) | 인바운드 및 아웃바운드: **모두 허용** (Allowed) |
| | 아웃바운드: **모두 허용** (Allowed) | |

---

이러한 핵심 개념들은 AWS 자격증 시험에서 자주 다루어지며, 특히 **보안 그룹과 네트워크 ACL의 차이점**은 중요하게 기억하실 필요가 있습니다.
